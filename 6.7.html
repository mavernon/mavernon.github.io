METHODS AND ENCAPSULATION
problem #   6.7
  1. class SuperClass {
  2. protected int i = 1;
  3. private void multiply(int m) {
  4. i *= m;
  5. }
  6. }
  7. public class Test extends SuperClass {
  8. public static void main(String[] args) {
  9. Test t = new Test();
  10. t.multiply(3);
  11. System.out.println(t.i);
  12. }
  13. }

Details ...

The scope of a method with a private access modifier is limited to the members of the class where the private method appears (i.e. in this example, only members of class SuperClass can access and interact with method multiply, located at line 3).

Therefore, attempting to call the private method multiply at line10, via the object instance t, will result in a compilation error (since the call takes place outside of the private method's scope ).

Access to a private method, or any private member, is not shared between a subclass and its superclass.
