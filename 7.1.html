An abstract class is not required to have any abstract methods.

However, a class that contains even one abstract method, must be labeled abstract.

The other circumstance when a class must be labeled abstract is, when it extends an abstract base class – but doesn’t implement any of the base class' methods. An abstract class similar to the one in this example (assuming that it were the subclass of an abstract super class) is an example of that.

Ultimately, any class that calls an abstract class (either through inheritance, or otherwise) must provide implementation code for the abstract class' methods (if they exist) – unless that class is itself labeled abstract.

An abstract class is typically designed to be an abstraction (e.g. similar to – but a more general version) of a concrete class. Moreover, while a concrete class can only extend one other concrete class, there is no restriction on how many abstract classes that they can extend (which provides concrete classes with a significant amount of flexibility).

Abstract classes cannot be directly instantiated. Despite that, it is possible (although somewhat counterintuitive) for an abstract class to even have a constructor (as shown at line 7).
