Details ...

Although a finally block (e.g. at line 11) will always execute ( ...with one exception – see example problem # 8.3), the code thats associated with a finally block may not always run to completion.

This point is illustrated in the example code above where, at line 13, the Integer variable i causes a NullPointerException to be thrown as a result of an attempt to use the variable (i) within an expression – while it has been identified as null (e.g. at line 4), without first initializing it to an actual value.
