The "|" symbol, that separates the two exceptions at line 7 and the two exceptions at line 10 is, in this case, not a bit-wise operator. But is instead, nothing more than a text separator, whose use (since its inclusion in Java 7) is meant solely as a short-cut that eliminates the need to write separate catch statements for each exception in a try-catch block.

There is a restriction to using this syntax â€“ the exceptions that are separated by this symbol cannot be related by inheritance. As a result, since ArithmeticException (at line 10) is a subclass of RuntimeException (at line 11), a compiler error is thrown at line 10 when an attempt is made to execute this code.

Note: The compiler error notwithstanding, the appropriate exception thrown when this code executes is ClassCastException, since the Long reference (i) at line 4 cannot be cast to an Integer (at line 5).
