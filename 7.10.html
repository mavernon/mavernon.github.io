Details ...

When two static methods have the same method signature (at lines 3 and 13 for example), overriding – wherein the subclass method takes precedence – does not occur (as it does when instance methods have the same signature). Instead, the static method thats being referenced at that particular time, is the method that takes precedence (e.g. the static method number, thats associated with the class A reference anInstance, at lines 10 and 11, takes precedence in this example).

As a result, the solution to this code is the int value 10 (from line 2), thats output when the println statement is invoked on line 11.
