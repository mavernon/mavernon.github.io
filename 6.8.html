The new keyword, at line 12, prompts a new object instance to be created from a default (i.e. no-argument) constructor â€“ but no default constructor is included in this class.

Under most circumstances, if no explicit default (no-argument) constructor exists, the java compiler will automatically create a default (no-argument) constructor (to facilitate compilation of the code, and enable a no-argument object instance to be built).

However, if the class of the object being instantiated already contains constructors - but none of them are default constructors (as in this case, at lines 2 and 5), then a default constructor will not automatically be added by the compiler. This results in an inability for the code to create a required (no-argument) object instance, causing code compilation to fail.

As noted, the class in this example includes two constructors that require a primitive argument (at lines 2 and 5), as well as, a method (at line 8) that returns no object. So, since there is no default (no-argument) constructor available (to facilitate building a new no-argument instance of the class, as required by the new keyword at line 12), the compilation of the code fails.
