Like the String class, classes in the java.time package are immutable and final; and, as a result, don't require the use of the new keyword in order to create an object instance.

The DateTimeFormatter class, of this package (line 3), provides three different types of formatters to use in formatting Date/Time output:

Predefined standard formatter's, identifiable by their usual assignment to a static constant (e.g. ISO_LOCAL_DATE_TIME, .ISO_LOCAL_TIME), are implemented by calling their format method (e.g. line 9).

Locale-specific formatter's (which use styles e.g. SHORT, MEDIUM) are used to present human-readable Date/Time output (e.g. lines 11 - 18).

The final approach to formatting Date/Time output, is to create your own format, by specifying a pattern (as shown on line 20 )

In any case, it is always necessary to identify an explicit reference to one of the java.time classes (e.g. LocalDate at line 7), understanding that the formatting approach chosen, must be applicable to the type identified.
