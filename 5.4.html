USING LOOP CONSTRUCTS
problem #   5.4
  1. public class Test {
  2. public static void main(String []args) {
  3. for (int i = 0; i < 10; i = 2) {
  4. if (i % 2 != 0) System.out.println(i);
  5. }
  6. }
  7. }

Details ...

After the initialization expression completes, the second expression in the for loop statement (at line 3), i < 10, evaluates to true, and enables the evaluation of the for loop target (the if statement) located on line 4.

The first time that the initial if statement logic in parenthesis is evaluated, the result is false. This halts any further processing of the if statement and forces the code flow to return to the for loop statement definition – where the third expression in the for loop statement definition ( which is designed to assign a new value to the variable used in the second expression in the for loop) automatically executes – before anything else occurs.

Following the execution of the third expression in the for loop, the code flow again returns to the second expression in the for loop statement, and evaluates it.

In this example, the third expression (at line 3) uses an assignment operator (as opposed to an increment or decrement operator) to assign the value of 2, to the variable “i”, and as a result, every time that the loop repeats - after the first for loop iteration, the variable "i" will always equal the value 2.

The result of this recurring, identical variable assignment is that, the for loop definition creates an infinite loop (i.e. the loop definition has no way of stopping the for loop from iterating).

This basically makes the loop target (i.e. the if statement, at line 4) irrelevant - since the code flow always returns to the for loop.

That said, if the for loop target (the if statement) did evaluate to True, then any directives or logic that followed the target (such as the println statement that follows the if at line 4) would be invoked (e.g. the value of the variable “i” would be printed at every loop iteration).
