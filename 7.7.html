Details ...

In this example, none of the classes has an explicit constructor. So a default constructor (thats never visible in the code) is automatically provided, to enable the creation of each object instance on lines 18 through 20, when the program code is compiled.

The statement at line 21, casts the object created at line 19, to class Z (via it’s object reference f2 ) with the intention of executing class Z's talk method. The cast executes without error, when the code is compiled.

However, the same cast returns a runtime error when the program code is actually executed – since, an object of class Y can never be the same (object) as an object of class Z.

In other words, although the compiler is able to detect that f2 references an object (i.e. "X") that has a legitimate relationship with object Z (i.e. "X" is the parent class of "Z") at line 21, the compiler is unable to recognize that no inheritance relationship exists between object Y (to which the reference f2 applies) and object Z. The JVM, however, does recognize this discrepancy – at runtime, and throws a runtime error then as a result.
