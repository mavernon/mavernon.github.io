An int array reference (i1), that contains five members, is created by the statement at line 3. Each of the five array reference members created, has the default value of zero. Similarly, an Integer array reference (i2) is created at line 4. However, each of the members of i2 are object types. These members have a default value of null.

While this code does compile successfully (with no type mismatch or syntax errors), a logical error (i.e. NullPointerException) is thrown at runtime. The runtime error occurs because, the expression following the if statement (at line 6) attempts to compare a null value - to an int value, when the code is run. In other words, the program code attempts to access the object located at array index location i2[2], which is null (i.e. it has no value) – and compare it to an int (primitive), having an integer value of zero.

Note that the java compiler, at compile time, doesn't have the ability to detect operational logic errors. This means that, in this case, it is perfectly legitimate for the if expression at line 6 to contain operands that have both a null - and a non-null value. However, what is not legitimate, is to perform an operation that compares them (which again, the compiler is unable to discern) – since the operands represent two incompatible object types.
