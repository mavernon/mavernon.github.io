The five members of int array i1, at line 3, all have the default value of zero. While the five members of the Integer array at line 4 all have the default value of null (since Integer is the object interpretation of int ).

Accordingly, the code compiles successfully; it contains no type mismatch or syntactical errors. However, a logical error (i.e. a NullPointerException) is thrown at runtime. The reason that the runtime error is thrown is, when the second if statement (at line 6) executes, the logic attempts to compare a null value, to an int value. In other words, the program attempts to access an object (i.e. i2[2] ) that has no value – and compare it to an int ( primitive) object that does have a value. This logic discrepancy is what causes the NullPointerException to be thrown at runtime.

Note that, the java compiler - at compile time, doesn't have the ability to detect logic errors; which means that, in this case, it is perfectly legitimate for the expression, i2[2] (at line 6) to have a null value, while the int to the right of it has the value of zero. However, what is not legitimate, is to compare them – since they represent two incompatible object types (which again, the compiler is unable to discern).
