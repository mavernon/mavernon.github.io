Although it seems counterintuitive that a class reference could be cast to an interface (as implied at line 12), it is possible – if the interface is related to the reference by inheritance or implementation. The mere existence of that possibility prevents a compiler error from being thrown, in this example.

However, the literal lack of an inheritance or implementation relationship between the class, and the interface in this example, means that an applicable cast does not exist (at line 12) and, as a result, a ClassCastException is thrown at runtime.

Below is an example (given a few modifications to this problem's original code) that shows how a class reference could be legally cast to an interface (e.g. through inheritance or implementation).

  1. ﻿class X {   
  2. private void print() {
  3. System.out.println( "X");
  4. }
  5. }
  6. ﻿class T extends X implements Y{   
  7. public void print() {
  8. System.out.println( "Y");
  9. }
  10. }
  11. interface Y {
  12. void print( );
  13. }
  14. public class Test {
  15. public static void main(String[] args) {
  16. X x = new T();
  17. ((Y)x).print();
  18. }
  19. }
