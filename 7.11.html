The interface constructor (line 2) is an illegal construct. Since interfaces cannot be instantiated, they similarly cannot have constructors. As a result, this code sample fails to compile (e.g. it produces a compiler error).

Interestingly, the abstract class constructor (at line 5) is a legal construct – even though an abstract class (also) cannot be instantiated . Specifically, the only way to invoke the constructor of an abstract class, is to access it through a subclass (of the abstract class) that can be instantiated.

In other words, if the interface constructor at line 2 were removed, and the code above resembled the sample code below, it would compile and execute without error (to include, the execution of the abstract constructor).

  2. }
  3. abstract class A implements X {  
  4. A() { System.out.print("A"); }
  5. }
  6. class B extends A {  
  7. }

  10. new Test();
