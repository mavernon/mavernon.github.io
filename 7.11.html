Details ...

The interface constructor (at line 2) is an illegal construct. The reason for this is, in addition to the fact that interfaces can't be instantiated, they similarly cannot have constructors. As a result, the code in this example simply fails to compile (e.g. it produces a compiler error).

Interestingly, the abstract class constructor (at line 5) is a legal construct – even though an abstract class (also) cannot be instantiated . The difference is, the constructor of an abstract class can only be invoked if its accessed through a subclass (of the abstract class) – that can be instantiated.

In other words, if the interface constructor at line 2 were removed, and the code above mirrored the sample code at the bottom of this page, it would compile and execute without error (to include, the execution of the abstract constructor at line 5).

Note: In the sample code below, class B (the subclass of abstract class A) is instantiated as a result of its inheritance relationship (at line 8) with class Test – when class Test is instantiated at line 10.

  2. }
