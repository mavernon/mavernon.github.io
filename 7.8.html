The code in this example uses objects (instead of primitives) as method types.

As a result, given the structure of this code, the only return type that can be used to declare method m1, at line 13, is either A or D.

The reason for this is, the return type of a method that overrides another method (e.g. the method at line 13 overrides the method at line 8) , must be the same type as the method it overrides (e.g. at line 8) -- or a subclass of that type (i.e. D is a subclass of A ... at line 4).

Note: If method m1 is declared as type A, at line 13, the return statement at line 14 must also be of type A (i.e. a D object can be an A, but an A object cannot be a D).

Also, the access modifier of the overriding method can be less restrictive than the access modifier of the method that it overrides (e.g. protected vs. default). Otherwise, the access modifiers for both methods must be the same.
