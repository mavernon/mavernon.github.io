METHODS AND ENCAPSULATION
problem #   6.5
  1. public class Test {
  2. public static int i = 2;
  3. public static void main(String []args) {
  4. int i = 8;
  5. Test anInstance = new Test();
  6. anInstance.multiply(i);
  7. System.out.println(i);
  8. }
  9. public void multiply(int i) {
  10. i = i * 3;
  11. }
  12. }

Details ...

In this example, a single variable identifier "i" is used as both a static variable at line 2, and a local variable at line 4. In other words, the same identifier is used within two different scopes (i.e. "i" has a different characteristic and value - that affects different segments of code - at different times). Managing this type of situation is done automatically, through shadowing (explained in the details of problem 6.2).

When this code is run, the statement at line 6, via the object instance created at line 5, calls method multiply (on line 9). The value of the local variable “i”, is passed as an argument, to fulfill the method’s parameter requirement. Yet, method multiply has a void return type. So, it doesn't return a value to the object instance (method caller) at line 6.

Meanwhile, shadowing, as noted earlier, ensures that the value of the local variable "i" is referenced at line 7, and not the value of the static variable "i" at line 2 (i.e. the static variable is hidden).
