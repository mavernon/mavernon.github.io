When multiple exceptions are caught, they must appear in order â€“ from the most specific exception to the most general one. Since RuntimeException is more specific than Exception, yet it appears as part of the last catch clause in this code (at line 12), this program will fail to compile.

If the exceptions in this example did appear in the correct order, RuntimeException would appear first (then Exception would appear, and then Throwable). Then, when the code executed, an ArithmeticException would be thrown after the last iteration of the for loop (due to an attempt to divide by zero). That ArithmeticException, since it inherits RuntimeException would be caught at line 8 (where the RuntimeException catch clause would be written), and the program code would be able to complete successfully.
