When classes have an inheritance relationship, the flow of logic between them travels via constructor chaining.

Constructor chaining occurs when the default constructor of a subclass, references the default constructor of it's super class . Alternatively, constructor chaining can occur if the default constructor of a subclass includes a constructor reference that is syntactically identical to a super class constructor (e.g. line 23).

The example code above combines constructor chaining with unboxing, and casting.

The base class at line 21 is chained to its super class at line 16, via the super class constructor reference at line 23 (in this case, unboxing from an Integer to a int type occurs simultaneously) . The constructor at line 16 is then chained to its super class reference - the default constructor at line 3.

Following the execution of the constructor at line 3, when the constructor at line 16 executes, the cast that occurs (at line 17) converts the value of the argument supplied to the constructor â€“ to an int.

Note: When unboxing and chaining occur simultaneously at the very beginning of this example, the reason that the constructor at line 16 is able to accept an int type as an argument is because, an int can be automatically converted to a double without any mathematical loss of precision.

The final result of this problem is the output of the String, "double", from line 18.
