When classes have an inheritance relationship, the flow of logic between them travels via constructor chaining.

Constructor chaining occurs when the keyword super is explicitly used as the first line in the body of a subclass (as is the case in this example – at line 23), with or without parameters, to reference the syntactically matching constructor of a super class. Alternately, constructor chaining also occurs when a subclass constructor explicitly (or implicitly, if no other class constructor exists) references the default constructor of it's super class (this is what enables the constructor at line 16 to be chained to its super class constructor at line 3).

This code in this example, also includes samples of unboxing and casting. Specifically, unboxing, from an Integer to a int type, occurs simultaneously where the subclass at line 21 is chained to its super class' constructor at line 16 via a reference to the super keyword (at line 23) – used in the body of the constructor at line 22.

Implicit conversion is what enables the constructor at line 16 to accept an int type (via the super constructor call, at line 23) as an argument. In other words, an int can be implicitly converted to a double without any mathematical loss of precision.

Following the execution of the constructor at line 3 (as constructor chaining reverses direction), when the constructor at line 16 executes, the cast that occurs (at line 17) converts the value of the argument supplied to the constructor – to an int.

The final result of executing this code, is the output of the String, "double", from the println statement at line 18.
