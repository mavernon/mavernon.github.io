In this example, the do portion of the do while loop statement contains (within it) both a for loop statement (at line 5), and a simple statement (at line 7).

The absence of an initialization expression in the for loop (at line 5) means that the variable i (in the for loop's second expression, i < 6) assumes the value of 4 (as a result of its initialization at line 3). Note that, the only for loop expression that absolutely must be present in a for loop statement, is the middle one â€“ the termination expression (which must always evaluate to either true or false).

When this code is executed, the for loop statement, at line 5, repeats two times (i.e. as long as the termination expression evaluates to true) before the loop terminates, and the code flow moves to the next statement (at line 7). Because the for loop (and the statement following it) are nested inside of the do portion of the do while, it always runs before the code in the while portion of the do while executes.

Notably, each time that the for loop statement in this example repeats, the value of the variable i is incremented by one (due to the post increment expression i ++ at line 5). This causes the value of i, when the for loop statement evaluates to false - and terminates, to be 6. However, this value is then immediately decreased by one (to the value of 5), after the final statement in the do portion of the do while (at line 7) executes.

When both statements, inside of the do portion of the do while loop, complete their iterations - and the code flow moves to the while portion of the do while loop at line 8, the while portion of the do while loop evaluates to false (i.e. 5 < 5 is false), causing the execution of the code to terminate.
