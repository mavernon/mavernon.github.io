  1. class A {
  2. int n = 3;
  3. A () {
  4. n = 4;
  5. }
  6. }
  7. public class Test extends A {
  8. int n = 2;
  9. public static void main(String[] args){
  10. A anInstance = new Test();
  11. System.out.println(anInstance.n);
  12. }
  13. }

Details ...

In this example, class Test has no explicit constructor. So a default constructor (thats never visible in the code) is automatically provided, when the new keyword prompts the creation of Test object, anInstance, at line 10.

The presence of the (invisible) default constructor in class Test, causes it to be automatically chained to the default constructor of the parent class, A (i.e. the flow of code transitions from one constructor to the next) . No output is generated when the parent class default constructor, and (then) the subclass constructor is called. However, the object created at line 10 explicitly references class A’s (obviously non-static) default constructor (i.e. the reference is, of type A), which ensures that the value of the default constructor’s local variable n is output via the println statement at line 11.

The instance variable n, at line 8, is ordinarily associated with any class Test object instance that is created — as long as that object is not created within a static context (unlike the object instance anInstance is, in this example).

For example, since the reference, anInstance, is executed within static method main (which is the only method available in class Test), the variable n (at line 8) has no possible associations -- i.e. if it were preceded by the word static, it would be a class variable, and able to be referenced (at line 11) with - or without being preceded by the name of the class.
