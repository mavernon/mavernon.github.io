The do portion of the do while construct in this code (lines 5 thru 8) ensures that the println statement (at line 6) will execute at least once. Directly following this action, the while portion of the do while (if it evaluates to true) enables any subsequent code logic to execute – before returning the code flow to the do portion of the do while (at line 5) where the process repeats.

In this example, the evaluation of the while portion of the do while (at line 7) requires a call to method m1 (at line 9). The logic inside of method m1 (i.e. the post-increment operator that follows the return statement) is designed to modify the value of a variable the very next time that the variable is referenced – after the post-increment operator is applied.

The next occurence in which the target variable i is referenced however, takes place outside of the scope in which the post-increment operator was applied. As a result, because the post-increment operator at line 10 has no impact on the target variable i (in this case) until after the variable leaves the scope in which the post-increment logic was initiated, the potential impact of the post-increment operator on variable i never occurs.

In other words, passing the value of variable i to the calling construct, via the return statement – coupled with a post-increment operator, ensures that the value that the variable has when it returns, is the same value it had when it arrived as an argument to the called method. That value, in this example, is zero.

The consequence of processing a variable whose value does not change, through the do while construct in this code (i.e. lines 5 thru 8) as described, is that an infinite loop occurs when the code is executed.

Conversely, if a pre-increment operator were applied to the value of variable i (e.g. ++ i) at line 10, the do while loop structure (at lines 5 thru 8) would cause the int value of c (as declared at line 4) to be printed five times, and the program would end.
