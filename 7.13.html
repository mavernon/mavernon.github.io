The object reference t, invokes its own method print at line 14 – since its super class contains an identical method (i.e. method overriding occurs). When the subclass reference t does this however, it’s method print calls the super class method print (via the use of the keyword super, at lines 16 and 17).

The println statement, contained in the super class method print, calls the object reference s – via the keyword this and, since the keyword this always refers to the object that currently calls it, the value of the reference s thats associated with the super class (at line 2) is returned. This results in the output “hello” when the code is run.
