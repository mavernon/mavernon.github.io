METHODS AND ENCAPSULATION
problem #   6.1
  1. public class Test {
  2. static int counter = 0;
  3. static void m1(String... a) {
  4. counter++;
  5. }
  6. public static void main(String []args) {
  7. m1 ();
  8. m1 (null);
  9. m1 (new String[] {"a", "b", "c"} );
  10. System.out.print(counter);
  11. }
  12. }

Details ...

When a class is compiled, the compilation process parses and identifies static members that are spontaneously executed (at runtime), generally before any other code processing occurs. This execution of static members doesn't require an object to be instantiated - or to even exist. A static method's execution is triggered by the availability of arguments that meet its definition criteria.

This means that, at runtime, static method m1 (on line 3) references any method with the same name and argument, and causes it to spontaneously execute - as if it were called directly. Since the static method uses a varargs as part of it's notation, under this scenario, each of the m1 methods in the main method (and their cast-able argument type) is a legitimate target, and gets invoked (without the need to instantiate an object).

Moreover, in this problem, every time that the static method references one of these target methods invoking it, the value of the class' static variable, counter, (which is mirrored in the body of the static method) increments by 1.

The final value of the static variable in this example (at line 10), is 3.
