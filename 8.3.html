The call (at line 4) by the Test object instance, to method number (declared at line 6) causes a RuntimeException object to be instantiated, at line 8 (note: instantiation simply creates an initial allocation of space in memory â€“ for the new object). However, no reference variable is declared, in association with the RuntimeException object, so the object is generally inert.

Meanwhile, the method's return statement (at line 10), because its located inside a finally block - and finally blocks always execute (with one noted exception), returns the int 1 to the method call at line 4 (i.e. the argument of the println statement).

The only time that a finally block will not execute is, if it is preceded by the statement: System.exit(0);
