This example code uses an enhanced for loop (at line 5).

Enhanced for loops include a variable (as part of the loop definition), that identifies/tags each member as it iterates through the loop target.

Unfortunately, the enhanced for loop in this example uses the incorrect syntax to declare the loop variable (at line 4), and as a result, attempting to execute this code fails -- and returns a compiler error.

The correct way to write an enhanced for loop (as shown below), is to include it's variable type within the loop definition (instead of separately, outside of the loop definition):

for (String letter : array)

In addition to inserting the correct syntax above at line 5, line 4 of the original sample should be removed as well - in order to successfully compile and run the code.

In a traditional for loop, the for loop target is written after the definition (for example, in problem 5.4 the if statement (target) is written after the for loop definition). On the other hand, when an enhanced for loop is implemented, the loop target (i.e. the String array in the code above) is written before the loop definition.
