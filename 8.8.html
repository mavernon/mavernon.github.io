Both classes extend Exception (an unchecked exception object) - although the two extended classes have no inheritance relationship with each other. Therefore ( since its implementation in Java SE 7), both subclasses must be included in the throws clause (at line 4).

... After the if statement at line 6 evaluates to true, Exception1 is thrown, and immediately caught (at line 11) – since (the compiler recognizes that) its a subclass of Exception. Following this, the exception is immediately re-thrown, but this time – not caught. The result is that, the exception is passed back down the stack to the calling method (e.g. main) where the program terminates with an exception (of the type defined at line 7).

The act of re-throwing an exception allows for inserting and executing a logging statement (or any another logic) just prior to the exception being re-thrown (i.e. just before line 12, in order to enhance the overall value of the source code (e.g. troubleshooting). This contrasts with throwing an exception and simply allowing a program to terminate without further action.

Re-throwing an exception, as described in this example, will only successfully compile if; 1) there are no other catch blocks that can handle the re-thrown exception, and 2) if the re-thrown exception is a subtype or supertype of the exception in the catch clause (at line 11).
