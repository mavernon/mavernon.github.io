The first target of the inner for loop at line 5, is the System.out.println statement (at line 6), which contains a reference to the variable "i". This is a bit deceptive and confusing, since "i" is initially introduced at line 4, as part of the definition of the first for loop ... seems more intuitive that a for loop definition/reference, and it's associated loop target, would be written adjacent to one another.

The fact that the inner for loop is nested within the outer for loop definition, provides a hint that the variable "i" is within the scope of the inner loop.

Ultimately, when the code flow returns to the inner for loop, after the first break at line 7 (at which point "i" equals 2) the deceptively located System.out.println loop target, ensures that the variable "i" is printed to output at least four times, for both values 2, and 3; before the code flow again is interrupted by the first break. Finally, the last if statement gets invoked after "i" is output as the value 5, three times, via the inner for loop, which then terminates the program.
