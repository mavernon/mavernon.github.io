The first while loop statement at line 4 contains two expressions, separated by the short-circuit operator &&. As a rule, the short-circuit operator, &&, will not evaluate the second part of a statement if the first part of the statement is false (as in this case). As a result, the first while loop statement (at lines 4 thru 5) fails to execute (i.e. it never iterates).

The behavior of the expression thats associated with the second while loop statement (at line 6) modifies the value of the variable i (via a post-increment operator) after the expression is evaluated. This repeats continually, until the value of i causes the while loop expression to evaluate to false (halting the while loop's ability to repeat). At that point, the code flow moves immediately to the statement at line 7; which outputs the last value that was incremented by the post-increment operator (i.e. 14) before the while loop statement at line 6 evaluated to false. Following this output, the program terminates.
