METHODS AND ENCAPSULATION
problem #   6.4
  1. public class Test {
  2. public static int a;
  3. public static void main(String []args) {
  4. new Test();
  5. }
  6. private Test() {
  7. int b;
  8. b = Test.a;
  9. System.out.println(a);
  10. }
  11. }

Details ...

When this code is compiled, the static int variable a is spontaneously initialized to it's default value of 0.

An instance of the class, initiated by the use of the new keyword at line 4, is then automatically created. The creation of this (unassigned) instance is modeled on the attributes of class Test' private default constructor (at line 6).

A key attribute of the private default constructor, is the statement at line 8 that assigns the value of the class' static variable (at line 2) to the local variable of the constructor.

The final solution to this code is that the System.out.println statement at line 9, returns 0 (i.e. the default value of the static variable a, declared at line 2).

Executing this code provides evidence that: 1) a private access modifier can be used with a constructor (in fact, using a private modifier is a way to encapsulate a constructor), and 2) it is possible to access a static member from a constructor.
