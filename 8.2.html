Including a throws clause in a method does not mean that an exception must be thrown. Instead, the throws clause is included to simply warn that its possible that executing the code will cause the specified exception to be thrown. For this reason, including the throws clause at line 3 has no adverse impact on this code (even though the exception specified in the throws clause is not actually thrown).
