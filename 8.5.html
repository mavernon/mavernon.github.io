  1. public class Test {
  2. public static void main(String []args) {
  3. try {
  4. int i = 10/0;

  7. System.out.println("catch");
  8. return;
  9. } finally {
  10. System.out.println("finally");
  11. }
  12. System.out.println("println");
  13. }
  14. }

Details ...

An exception is thrown (due to an attempt to divide by zero, at line 4) and caught by the catch block (at line 6) – causing any code inside of the catch, except for the return statement, to execute - before the code flow is then routed to the finally block (at line 9).

Since the finally block always executes, any code inside of the finally (i.e. the statement at line 10), similar to the code inside of the catch block, also runs. After the code in the finally block completes, the flow of the code is then transfered to the return statement at line 8 – inside of the catch, where it then gets routed back to the calling method (main) before terminating.

Consequently, the result of the code flow in this example is that, neither the code statement at line 4 – or at line 12 executes.

Interestingly, inserting a return statement inside of the finally block, would make it impossible for the code at line 12 to ever run – resulting in an unreachable statement compiler error.

On the other hand, as currently written, it's entirely possible to add additional code to this example, that causes the statement at line 12 to execute – even after the code flow returns to the calling method. For this reason, no unreachable statement compiler error is thrown in this example, even though the statement at line 12 (and 4) never executes, since the statement is still reachable.
