Details ...

Although the letter "s" identifies both the static (i.e. class), and local parameter variable (at lines 2 and 5), the variable with the more limited scope, the local variable, takes precedence.

A majority of the time that this type of duplication (i.e. shadowing) occurs, it occurs unintentionally. In any case, the variable with the limited scope casts a shadow over the larger scope variable, leaving the static variable unable to be seen (i.e. hidden).

However, shadowing can be overridden simply by referring to the shadowed variable using its fully-qualified name.

For example, modifying the parameter variable by writing: System.out.print (aCopy.m1(Test.s)) at line 6, will execute the statement using the static variable "s" (from line 2) instead.
